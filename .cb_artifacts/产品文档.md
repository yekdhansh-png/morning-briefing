## 一、产品定位

### 一句话介绍

> **"7:30 看完不踏空"** —— 把券商研究所、财经媒体、外盘行情、政策文件汇成一份 3 分钟可读完的盘前早报，每个交易日清晨自动推送。

### 目标用户

- A 股散户投资者 / 兼职炒股的上班族
- 没时间通读整份券商晨报、又不想错过盘前关键信号
- 希望"读完就能下单"，而不是读完还要自己消化

### 核心价值

| 痛点 | 我们的解法 |
|---|---|
| 早报来源散：财联社、证券时报、华尔街见闻、上证早知道，每天 5+ 个 App | **一站聚合 4 路信源 + 美股收评 + 上证早知道** |
| 信息量大但读不完：散户没时间筛 100+ 条 | **LLM 选 TOP 3 要闻 + TOP 2 利好**，每条 30-60 字解读 |
| 看完还不知道买什么：要闻 ≠ 标的 | **每个利好自动配 2 只受益股**（龙头 + 弹性搭配） |
| 不懂"今天会涨什么"：缺方向判断 | **市场温度** + 板块映射 + 强度评级 |

### ④ 重磅要闻 · TOP 3 ★★ 已自动化

#### Step 1 — 信源（3 路并发抓取）

| 信源 | 接入方式 | 当日典型量 |
|---|---|---|
| **财联社早报** | HTTP 抓 `cls.cn/subject/1151` → 详情页 | 35-40 条 |
| **证券时报早知道** | API `egs.stcn.com/news/intelligence.html?_format=json` | 5-10 条 |
| **华尔街见闻早餐** | API `api-prod.wallstreetcn.com/.../articles` | 0-3 条（有反爬） |

**合计约 40-50 条候选池**，每条含标题 + 正文 + 章节 + 来源。

#### Step 2 — LLM 选 TOP 3（卖方研究员评分模型）

**4 维度评分**（A/B/C/D 加权，权重 0.30/0.25/0.20/0.25）：
- **A 直接性**：政策细则 / 龙头公告 / 大单事件 > 模糊预期
- **B 广度**：板块级影响 > 个股级
- **C 稀缺性**：独家 / 首发 > 已被市场充分讨论
- **D 散户必知度**：影响开盘走向 > 长线影响

**5 条专家规则**（覆盖常见翻车点）：
- 宏观重磅事件加成
- 小盘股过滤
- 避雷词（"专家预测"/"小作文"等）
- 数据类降权
- **时效性铁律 / 边际定价**（最重要）

**时效性 4 档**：
- ★★★ 凌晨 / 早盘前事件 → A 股**未计价**，头条候选
- ★★ 昨日收盘后（15-24:00） → 今日首次反应
- ★ 昨日盘中（09:30-15:00）→ 视尾盘走势续作
- ✗ 24 小时前的旧闻 → A ≤ 4 降级

例：选英伟达凌晨财报 > 中美元首昨日上午会谈（虽然规模大但已被消化）

#### Step 3 — LLM 写解读（卖方风格三步翻译法）

每条新闻输出：
- **title 标题**：40-55 字，纯事实陈述（**17 词黑名单**禁用"有望/景气度持续/映射/战略协作深化"等判断词）
- **signal 核心信号**：1 句话 30 字，给方向判断
- **signalDetails 三段**：[事实路径] → [受益方向] → [风险点]
- **affectedName 影响板块**：≤5 字，**必须是 A 股可交易概念**（禁"能源链/外贸链/宏观"等泛词）
- **affectedDirection**：正面 / 负面
- **strength 影响强度**：强 / 中 / 弱

#### 程序兜底
- 黑名单词扫描：title 含禁用词 → stderr 打 WARN
- 标题字数硬约束 40-55

### ⑤ 利好催化 · TOP 2 ★★ 已自动化

#### Step 1 — 多路信源合并

| 信源 | 内容 |
|---|---|
| **复用早报候选池** | 财联社 + 证券时报 + 华尔街见闻（40-50 条） |
| **新增上证早知道**（SNP） | 上海证券报 7:01 发布的卖方研究院风格早报 |

**上证早知道章节解析**（6 章节 + 权重）：
- 上证聚焦 (1.0) / 产业情报 (1.0) — 强催化
- 公司新闻 (0.6)
- 上证精选 (0.4)
- 资金观潮 (0.3)
- 今日提示 (0.1)

#### Step 2 — LLM 选 TOP 2 利好（5 层评估）

| 层 | 维度 |
|---|---|
| **第 1 层** | 去重：与要闻 TOP 3 主题不重复（硬过滤） |
| **第 2 层** | 热门赛道白名单（AI 算力/机器人/绿电/光通信 等），冷门淘汰 |
| **第 3 层** | 边际定价（凌晨/盘后 > 24h+ 旧闻） |
| **第 4 层** | **信源等级 S/A/B/C/D**（详见下表） |
| **第 5 层** | **行业基本面信号**（价格 / 供需 / 业绩 / 拐点 / 需求） |

**信源等级金字塔**（决定催化"含金量"）：

| 级 | 来源类型 | 例子 |
|---|---|---|
| **S** | 官方文件强制力 | 发改委发文 / 央行政策 / 央地联合发文 |
| **A** | 龙头公司公告 | 璞泰来 56 亿投建 / 业绩超预期 |
| **B** | 海外产业链确认 | 谷歌 I/O 大会 / 英伟达财报数据 |
| **C** | 头部机构观点 | 中金 / 大摩 / 华泰研报 |
| **D** | 媒体传闻 | "据传 / 接近 X 的人士"，基本不选 |

**约束**：TOP 2 中**必须至少 1 条 S 或 A 级**，避免两条都是弱催化。

#### Step 3 — 输出字段（与设计稿严格对齐）

```json
{
  "concept": "绿电",            // 概念名 ≤6 字（前端红色 chip）
  "tag": "政策聚焦",            // 6 选 1 白名单
  "event": "国家发改委、能源局发布多用户绿电直连通知",  // 事件 ≤25 字
  "catalystPath": "政策落地→绿电消纳价值重估→运营商估值修复",
  "catalyst": "完整催化逻辑 30-50 字",
  "event_level": "S",
  "fundamental_signal": "..."
}
```

**6 个标签固定白名单**（程序兜底自动 fallback）：
1. **政策聚焦** — 部委发文 / 监管 / 试点 / 国家间合作
2. **机构热评** — 券商研报 / 评级 / 调研
3. **市场热议** — 资金 / 龙虎榜 / 北向
4. **资产映射** — **仅大宗商品价格**（LME / COMEX / 金银铜油气煤铁矿稀土）
5. **涨价信号** — A 股产业链产品涨价 / 供需缺口 / 出口管制
6. **景气赛道** — 订单 / 业绩 / 周期 / 海外巨头新品发布

#### Step 4 — 每个利好选 2 只受益股（5 道防线绝不开天窗）

**两层选股逻辑**：
- **第 1 层 直接受益（必须）**：龙头 / 业务占比高 / 产业链关键位置
- **第 2 层 股性活跃（加分）**：5/20 日涨幅适中 / 主力净流入 / 中盘弹性 / 龙头属性

**5 道兜底防线**：
1. westock search 拉相关股票（最多 30 只）
2. concept_leaders 兜底字典合并（30+ 主线赛道，80+ 龙头股，手动维护）
3. 并发拉实时数据：5/20/60 日涨幅 + 主力 5 日资金 + 行业分类
4. LLM 选 2 只 + 写理由
5. 代码校验失败 → 重试 1 次 → 终极兜底从字典挑

**搭配原则**：一龙一弹（龙头 + 弹性次新），避免两只同等定位。

**reason 写作规则（防自爆）**：
- 实时数据是**可选项**，正面才写（"主力 5 日净流入 12.5 亿"✅）
- 严禁写负数 / 流出 / 跌幅（防止"主力 5 日 -17.9 亿"这种自爆）
- 没有正面数据时改用：业务地位 / 业绩兑现 / 产业地位 / 题材属性 / 机构筹码

### 技术栈

| 层 | 选型 | 理由 |
|---|---|---|

| 设计点 | 实现 |

## 五、亮点 & 突破

### 1. 不是"喂模型，让它说"，是"卖方研究员脑子拆给模型用"

每个 LLM Prompt 都是一份**SOP 文档**，定义：
- 必填字段 + 字数硬约束
- 黑名单词（防判断词污染标题）
- 正反例对照（用真实翻车场景做锚定）
- 程序兜底（不依赖模型的"自觉")

举例 — 要闻 prompt 里的"时效性铁律"：
```
反例：选了"中美元首昨日上午会谈"（已发酵）✗
正例：选凌晨英伟达财报（未计价）✓
```

### 2. 信源多路冗余 + 主备 fallback

| 模块 | 主信源 | 备信源 | 兜底 |
|---|---|---|---|
| 外盘指数 | westock | — | 上次数据 |
| 大宗商品 | 新浪外盘期货 | — | 上次数据 |
| 早报新闻 | 财联社 + 证券时报 + 华尔街见闻 | — | 任一可用即工作 |
| 利好催化 | 上证早知道 + 早报池 | — | 任一可用即工作 |
| 选股 | westock search | concept_leaders 龙头池 | LLM 失败时直接挑字典 |

**任何一路源挂掉都不影响整体出版**。

### 3. 数据传导链 100% 可解释

每条 LLM 输出都能追溯到：
- 原信源章节（`source_section: "上证早知道·上证聚焦"`）
- 原始标题（`source_title`，verbatim 保留便于回溯）
- 信源等级（`event_level: S/A/B/C/D`）
- 基本面信号类型（`fundamental_signal`）

读者 / 风控 / 我自己想知道"为什么选了这个"，**5 秒可追源**。

### 4. 视觉 1:1 还原设计稿

- A 股配色铁律：涨红 `#E54D42` / 跌绿 `#1FAE6F`
- NO.01/02/03 徽章 / 5 格强度虚线 / 影响资产 3 栏布局（左板块/中胶囊/右强度）
- 跨平台一致性：emoji → SVG icon（避免不同 OS emoji 渲染差异）
